# 業績入力フォーム　マニュアル（入力者むけ）

> 対象：研究室のメンバー（学生・研究員など、自分の業績を登録する人）
> このマニュアルだけ読めば登録できます。専門知識は不要です。

---

## 1. これは何をするフォームですか？

あなたの研究業績（論文・発表・受賞など）を、**1 つの Google フォームから登録**するためのものです。

- 送信した内容は、研究室の業績データベースに自動で保存されます。
- 保存された業績は、担当者（教員・スタッフ）が確認したあと、業績まとめアプリで使われます。
- **入力はこのフォームから送るだけ**。表計算ソフトやプログラムは一切さわりません。

```
あなた：リンクを開く → 種別を選ぶ → 出てきた欄を入力 → 送信
   ↓（自動）
データベースに保存（担当者が確認して反映）
```

---

## 2. 準備するもの

- **フォームのリンク（URL）1 本だけ**
  　研究室の担当者から、メール・チャットなどで配られます。
  　（リンク例：`https://docs.google.com/forms/d/e/............/viewform`）
- **Google アカウントでのログイン**
  　送信時にメールアドレスの記録を求められることがあります。普段お使いのアカウントで OK です。

> **ヒント：** リンクが分からないときは、担当者（**宮崎／倉光**）にたずねてください。

---

## 3. 登録のながれ（5 ステップ）

### ステップ 1　リンクを開く
配られた URL をクリックします。ブラウザ（Chrome / Safari / Edge など）でフォームが開きます。

### ステップ 2　報告者氏名を入れる
いちばん上の **「報告者氏名」** に、あなたの名前を入力します。
- 例：`山田 太郎`
- 名簿どおりの表記がおすすめですが、多少のゆれは後で自動で吸収されるので心配いりません。

### ステップ 3　業績種別を 1 つ選ぶ
**「業績種別」** で、登録したい業績の種類を **1 つだけ** 選びます。
選ぶと、その種類に必要な入力欄だけが自動で表示されます。
（種類の選び方は「4. どの種別を選べばいい？」を参照）

### ステップ 4　出てきた欄を入力する
表示された欄を上から順に入力します。
- **「* 必須」と書かれた欄は、必ず入力**してください（空だと送信できません）。
- それ以外（任意）は、分かる範囲で入れれば OK。空欄のままでも送信できます。
- 各欄の下に **記入例** が出ています。迷ったら例のとおりに入れてください。

### ステップ 5　送信する
いちばん下の **「送信」** ボタンを押して完了です。
- お疲れさまでした。これで登録は終わりです。
- **別の業績も登録したいとき**は、もう一度リンクを開いて、ステップ 2 から繰り返します。
  　（1 回の送信で登録できるのは 1 件です）

> **※** 送信したあと、すぐにアプリに出てこなくても正常です。担当者が内容を確認してから反映されます。

---

## 4. どの種別を選べばいい？

| 選ぶ種別 | こんな業績のとき |
|---|---|
| **原著論文・英文総説** | 学術雑誌に載った論文、英語の総説（レビュー） |
| **著書・和文総説** | 本・教科書の章を書いた、日本語の総説 |
| **発表・講演** | 学会発表、口頭発表、ポスター発表、招待講演 |
| **受賞** | 学会賞、奨励賞などを受けた |
| **アウトリーチ** | 市民講座、出前授業、一般向けの活動 |
| **広報・パブリシティ** | 新聞・TV・Web・雑誌などで紹介された |

> 迷ったら、いちばん近いものを選べば大丈夫です。

---

## 5. 種別ごとの入力欄（何を入れるか）

「★」が付いた欄は **必須** です。それ以外は分かる範囲で。
タイトルや雑誌名は **日本語と英語の両方の欄** があることがあります。**分かる方だけでも構いません**（両方入れると、表示言語に合わせて自動で使い分けられます）。

### 5-1. 原著論文・英文総説
- ★ 発行日（例：`2026/04/10`）
- ★ 区分（原著論文 / 英文総説）
- ★ 査読の有無（査読あり / 査読なし）
- ★ 著者（**記載順に全員**。例：`Yamada T, Hayashi N, Suzuki I`）
- 論文タイトル（日本語・任意）
- ★ 論文タイトル（English）
- 雑誌名（日本語・任意）／ ★ 雑誌名（English・正式名称、例：`Scientific Reports`）
- 略誌名（日本語・任意 ／ English、例：`Sci. Rep.`）
- 巻 / 号 / ページ（例：ページ `871` または `12-20`）
- DOI（例：`10.1038/s41598-025-32402-2`）
  　→ **DOI を入れると、英語タイトル・雑誌名・巻号ページなどを自動で補ってくれます**（空欄だけ埋まります）。分かるなら入れるのがおすすめです。

### 5-2. 著書・和文総説
- ★ 発行日
- ★ 国内 / 国際
- 査読の有無（任意）
- ★ 著者（例：`山田 太郎, 鈴木 一郎`）
- 章・総説タイトル（日本語 ／ English・任意）
- ★ 書名（日本語、例：`最新・睡眠科学`） ／ 書名（English・任意）
- 章 / 編者 / 巻 / 号 / ページ / 出版社（例：`医学書院`）
- DOI / ISSN / ISBN（例：`978-4-260-00000-0`）
  　→ **ISBN を入れると、書名・出版社・出版日を自動で補ってくれます**。

### 5-3. 発表・講演
- ★ 発表日
- ★ 国内 / 国際
- ★ 演題（日本語） ／ 演題（English・任意）
- ★ 発表者（**連名は記載順に全員**。例：`山田 太郎, 林 直子`）
- ★ 学会・研究会名（日本語） ／ 学会・研究会名（English・任意）
- シンポジウム名（日本語 ／ English・任意）
- 招待の有無（招待あり / 招待なし）
- 開催地（例：`横浜`）
- 発表形式（口頭 / ポスター / 口頭＆ポスター）

### 5-4. 受賞
- ★ 受賞日
- ★ 国内 / 国際
- ★ 受賞者（例：`山田 太郎`）
- ★ 賞の名称（日本語） ／ 賞の名称（English・任意）
- 受賞対象の研究・業績（任意）
- 授与団体（例：`日本睡眠学会`）

### 5-5. アウトリーチ
- ★ 実施日
- ★ 国内 / 国際
- ★ 実施者（例：`山田 太郎`）
- ★ 活動概要（日本語、例：`市民講座「睡眠のふしぎ」`） ／ 活動概要（English・任意）
- 開催地・媒体（任意）

### 5-6. 広報・パブリシティ
- ★ 掲載日
- ★ 媒体種別（新聞 / TV / Web / 雑誌 / その他）
- ★ 媒体名（例：`○○新聞`）
- ★ 掲載人物（例：`山田 太郎`）
- ★ 掲載概要（日本語） ／ 掲載概要（English・任意）
- リンク（URL・任意）

## 6. 入力のコツ（ここだけ読めば失敗しません）

### 著者・発表者の書き方
- **記載順に全員**を書きます（自分だけではありません）。
- 区切りは **カンマ（,）** が分かりやすいです。
  　例：`Yamada T, Hayashi N, Suzuki I` / `山田 太郎, 林 直子`
- 表記のゆれ（`Hayashi N` と `林 直子` など）は、システムが自動でまとめるので**気にしなくて大丈夫**です。
- 発表者を示す氏名の先頭の丸印（○●◎ など）は付けても消されます。付けても付けなくても OK です。

### 日付の書き方
- `2026/04/10` のように **年/月/日** で入れるのが確実です。
- カレンダーから選ぶ欄では、表示どおりに選べば OK です。

### 日本語・英語の欄
- 両方ある欄は、**分かる方だけ**でも構いません。
- 両方入れておくと、日本語表示・英語表示のどちらでもキレイに出ます。

### DOI・ISBN は分かれば入れる
- 論文の **DOI**、本の **ISBN** を入れると、残りの情報を**自動で埋めてくれます**。
- 手入力の手間が減るので、分かるときはぜひ入れてください。

## 7. たくさんまとめて登録したいとき（任意・上級）

業績がたくさんあるときは、**「まとめて貼り付け（複数件を一度に）」** が便利です。

1. 業績種別の選択で **「まとめて貼り付け」** を選ぶ。
2. **貼り付ける業績の種別を 1 つ** 選ぶ（1 回の送信につき 1 種別。論文と発表は分けて送る）。
3. researchmap などからコピーした業績リストを、そのまま貼り付ける。
   - **1 件は「タイトル → 著者 → 誌名/学会名＋日付」の並び**で書きます。
   - **行の末尾に日付がある行（例：`2025年7月` / `Jul, 2025` / `2025/07`）が、1 件の区切り**になります。
   - 1 項目は 1 行で、余計なメモは入れずに貼ってください。
4. 送信すると、複数件がまとめて登録されます。

> **注意：** 自動で読み取るため、まれに分け方を間違えることがあります。件数が多いとき向けの機能です。1〜数件なら、通常の入力（ステップ 1〜5）が確実です。

## 8. よくある質問（FAQ）

**Q. 送信したあと、内容をまちがえました。直せますか？**
A. はい、直せます。フォームからは送信後に直せませんが、**データベース（スプレッドシート）の自分の行を直接編集して構いません**。
　- 自分が報告した行（`submitter` があなたの名前の行）を見つけ、**データの欄だけ**を直してください（列名や `status` の欄は編集できないように保護されています）。
　- 修正すると、その行は自動で **「未確認」に戻ります**。これは正常な動作なので、**そのまま編集して保存すれば大丈夫**です（担当者があらためて確認します）。
　- データベースの場所や編集権が分からないときは、担当者（**宮崎／倉光**）にたずねてください。

**Q. 必須欄が分からず、送信できません。**
A. 「* 必須」と赤く表示される欄が空のはずです。画面を上にスクロールして、未入力の必須欄を埋めてください。

**Q. 送信したのにアプリに出てきません。**
A. 正常です。担当者が内容を確認（承認）してからアプリに反映されます。少し時間をおいてください。

**Q. 同じ業績を 2 回送ってしまいました。**
A. システムが重複に気づいて目印を付けます。担当者が確認時に整理するので、放っておいて大丈夫です。

**Q. 著者名のスペルや表記に自信がありません。**
A. 多少のゆれは自動で吸収されます。分かる範囲で正確に入れてください。

## 9. 困ったときは

担当者は **宮崎／倉光** です。

- リンクが開けない／分からない → 宮崎／倉光へ。
- 入力内容の修正（自分で直せないとき・編集権が分からないとき）→ 宮崎／倉光へ。
- それ以外の不明点 → 宮崎／倉光へ。

> このフォームの管理・運用については、担当者むけの「管理者マニュアル」があります。
