# 業績入力フォーム（1 フォーム＋自動取込）作成手順

各メンバーが **1 つの Google フォーム** で業績を送信 → 回答が **自動で** 業績 DB（Canonical スプレッドシート）の該当タブへ蓄積 → 本アプリ（研究業績サマライザー）で抽出・整形する運用にするためのガイド。

> 以前は「種別ごとに 6 フォーム」を作る設計でしたが、**リンクが 6 本に分かれて使いにくい／作成・保守が重い／回答を DB へ手で転記する手間が残る** という問題がありました。
> 本手順は **フォーム 1 つ・配布リンク 1 本** に統合し、送信から DB 追記までを Apps Script で自動化します。

---

## 全体像

```
メンバー   1 本のリンクを開く → 先頭で「業績種別」を選ぶ → その種別の項目だけ入力 → 送信
   │
   ▼ （Apps Script: onFormSubmit トリガが自動起動）
業績 DB    Canonical スプレッドシートの該当タブへ 1 行追記（status=未確認, source=form）
   │
   ▼ （あなた＝キュレーターが内容を確認）
反映       status を「確認済」に変えた行だけ、アプリに表示される（品質ゲート）
```

- **メンバーの手間**：リンク 1 本。種別を選ぶと、その種別に必要な欄だけが出る。
- **あなたの手間**：フォーム作成はスクリプトを 1 回実行するだけ。日々は「未確認 → 確認済」を確認するだけ
  （後述の**キュレーター・メニュー**でワンクリック一括承認できる）。
- **持続可能性**：フォームの定義はスクリプト（`scripts/forms/publication_form.gs`）にあり、項目を変えたら再生成できる。手作業のセクション組みや回答の手転記は不要。

### 自動でやってくれること（送信時）
- **英日両方入力**：タイトル・雑誌名/学会名などは日本語と英語の両欄を用意。両方入れればアプリの表示言語で自動選択、片方だけなら有る方を使う。
- **DOI 自動補完**：論文・著書で DOI を入れて送ると、CrossRef を参照して英語タイトル・雑誌名・巻号ページ・発行日の**空欄だけ**を補完する（`note` に `crossref`）。
- **重複フラグ**：同じ DOI、または同じ日付＋タイトルが既にあると、`note` に `dup_of=<既存ID>` を付けて**残す**（消さない）。あなたが見て削除・統合を判断。

> ⚠️ これらを有効にするには、DB が **v2 スキーマ**（`_ja`/`_en` 列＋`note` 列）である必要があります。
> 既存 DB は [`migration-guide.md`](migration-guide.md) の「DB を v2 へ上げる」を一度だけ実施してください。

### メンバー自身が「まとめて貼り付け」で一括登録（CLI 不要）
種別選択の最後にある **「まとめて貼り付け（複数件を一度に）」** を選ぶと、貼り付け用セクションへ進みます。
メンバーは **researchmap 等からコピーしたプレーンテキスト**を貼って送信するだけで、複数件が一度に登録されます。

- 「貼り付ける業績の種別」を 1 つ選ぶ（1 回の送信につき 1 種別。論文と発表は分けて送信）。
- 「業績リスト」に貼り付け。1 件の並びは **タイトル → 著者 → 誌名/学会名＋日付**。
  **末尾に日付がある行（例: `2025年7月` / `Jul, 2025` / `2025/07`）が 1 件の区切り**です。
- 送信すると Apps Script が解析し、巻号・ページ・日付を自動抽出、タイトル/誌名/学会名は本文の言語で
  `_ja`/`_en` に自動振り分けて、各件を `status=未確認` で追記（重複は `dup_of` フラグ）。
- 解析の都合上、**1 項目 1 行・余計な注記なし**で貼ってください（researchmap のコピーそのまま）。

> CLI（`scripts/ingest_paste.py`）でも同じ取り込みができますが、**メンバー配布にはこのフォームの貼り付けが最も簡単**です。

> **設計方針（DOI を必須にしない）**：論文以外（発表・受賞・著書・アウトリーチ・広報）に DOI は無く、和文誌・和書は DOI 解決が弱いため、全項目を手入力で受けます。DOI は論文・著書の **任意項目**（形式検証つき）に留めます。

## 準備するもの

- 業績 DB となる **Canonical スプレッドシート**（種別ごとのタブを持つ。作り方は [`migration-guide.md`](migration-guide.md) を参照）。その **スプレッドシート ID** を控えておく（URL の `/d/` と `/edit` の間の長い文字列）。
- 上記スプレッドシートを **編集できる Google アカウント**（＝あなた）。スクリプトはこの権限で DB に書き込みます。アプリ側の「閲覧のみ公開」はそのままで構いません。

## 作成手順（1 回だけ）

### 1. Apps Script プロジェクトを作る
1. <https://script.google.com> を開き、右上が業績管理用の Google アカウントか確認。
2. **「新しいプロジェクト」** をクリック。
3. 既定の `コード.gs` の中身を**全部消す**。

### 2. スクリプトを貼り付ける
1. リポジトリの **`scripts/forms/publication_form.gs`** を開き、**全文をコピー**。
2. 手順 1 で空にしたエディタへ**貼り付け**。
3. 上部の保存アイコン（💾）で保存。プロジェクト名は「業績登録フォーム」等に。

### 3. 書き込み先 DB を指定する
貼り付けたコードの先頭付近にある次の 1 行を探し、ダブルクォートの中身を**自分の Canonical スプレッドシート ID** に書き換えて保存：

```javascript
var CANONICAL_SHEET_ID = "1_7b_-6EsRNr6tW1naDj8QJ5M0espE5Wv";
```

> ここが**唯一**書き換える箇所です。種別ごとの設問は `FIELD_MAP` に定義済みで、`schema.py` と一致しています（`tests/test_form_fields.py` が自動チェック）。

### 4. フォームを生成する（認可）
1. エディタ上部の関数選択で **`buildForm`** を選ぶ。
2. **「実行」** をクリック。
3. 初回は認可ダイアログが出ます → **自分のアカウントを選択** →「詳細」→「（プロジェクト名）に移動」→ **許可**。
   （Google フォームの作成・スプレッドシートへの書き込み・トリガ設置を許可します。）
4. 実行が終わると、下部の**実行ログ**に次が出ます：
   ```
   公開用 URL（メンバーへ配布）: https://docs.google.com/forms/d/e/.../viewform
   編集用 URL（あなた専用）:   https://docs.google.com/forms/d/.../edit
   ```
   ログが見えないときはメニュー **表示 → ログ**（または `Ctrl+Enter`）。

> ⚠️ `buildForm` を**もう一度実行すると新しいフォームが別に作られます**。やり直したいとき以外は再実行しないでください（設問を増減したいときは `FIELD_MAP` を直してから作り直す運用）。

### 5. 動作をテスト送信で確認する
1. **公開用 URL** を開き、種別を 1 つ選んで 1 件テスト送信。
2. Canonical スプレッドシートの該当タブ（例：`Original Papers`）を開き、最下行に
   `status=未確認`・`source=form`・`record_id=PAP-xxxx` の行が増えていれば成功。
3. その行の `status` を **`確認済`** に変更 → アプリを開き、件数に反映されることを確認。
4. 確認できたらテスト行は削除して構いません。

### 6. メンバーへ配布する
**公開用 URL** をメール・チャット等で共有します。リンクはこの 1 本だけです。
フォームの説明欄（生成時に設定済み）に、記入例・締切・問い合わせ先を追記しておくと親切です。

## 確認をラクにする：キュレーター・メニュー（`canonical_tools.gs`）

確認作業（未確認→確認済、DOI 補完、重複チェック）をスプレッドシート上のメニューから行えます。

1. **Canonical スプレッドシート**を開く →「拡張機能」→「**Apps Script**」。
2. `scripts/forms/canonical_tools.gs` を全文コピペして保存。
3. スプレッドシートを再読み込みすると、上部に **「業績DB」メニュー**が出る（初回操作時に認可）。

メニュー項目（record タブで行を選択してから実行）:
- **選択行を承認(確認済)** … 選んだ行の `status` を一括で `確認済` に（＝アプリに反映）。
- **選択行を未確認に戻す** … `status` を `未確認` に。
- **選択行を DOI で補完** … その行の DOI から CrossRef で空欄を補完（論文・著書）。
- **このタブの重複を再チェック** … 重複する後発行に `note=dup_of=<既存ID>` を付ける。
- **色分けを設定（全タブ）** … 重複を再判定してから、未確認の行を**黄背景**、重複候補（`note` に `dup_of`）を**赤の太字**にする条件付き書式を設定。確認して `確認済` にすると黄が消える。
- **一括下書き: シートを準備 / 表に展開 / 取り込む** … 下記「貼り付け→表で確認→取り込み」を行う。

### 一括下書き（貼り付け → 表で確認・修正 → 取り込み）
researchmap 等の生テキストを、**表形式で DB 列を確認・編集してから**取り込みたいときに使います（Google フォームでは表編集ができないため、スプレッドシート上で行います）。

1. 「業績DB →**一括下書き: シートを準備**」… `一括下書き` シートが作られる。
2. **B1 で種別**を選び、**A4 以降に生テキストを貼り付け**（1 行＝1 行。末尾に日付がある行が 1 件の区切り）。
3. 「業績DB →**一括下書き: 表に展開**」… 右側（C 列〜）に **DB 列（`date`/`title_ja`/`title_en`/… ）の表**が出る。巻号・ページ・日付は自動抽出、タイトル等は日英へ自動振り分け、`査読の有無` 等は**プルダウン**。
4. 表のセルを**確認・修正**（足りない英語名や区分を選ぶ等）。
5. 「業績DB →**一括下書き: 取り込む**」… 該当タブへ `status=未確認` で追記（重複は `dup_of` フラグ）。下書きはクリアされる。
6. いつもどおり内容を確認して `確認済` に。

## 日々の運用（未確認 → 確認済）

- フォーム送信は **`status=未確認`** で DB に積まれ、アプリには出ません（下書き扱い）。
- 内容を確認したら、**「業績DB」メニュー → 選択行を承認(確認済)** でまとめて承認（または `status` を手で `確認済` に）。これだけがアプリへの反映条件です（品質ゲート）。
- **著者名の表記ゆれ**は気にしなくて構いません。名簿（`Input confirmation`）との名寄せをアプリ側（`roster.py`）が吸収します。
- **重複**は `note` の `dup_of=...` が目印。確認時に行を削除するか `未確認` のままにしてください。

## 限定公開（学外に出さない）

- フォーム設定（⚙）→「**回答**」→「**回答を 1 回に制限する／組織内に限定**」を必要に応じて ON（同一 Google Workspace 組織のみ・ログイン必須にできる）。
- 個人 Google アカウント運用で組織制限が使えない場合は、**リンクを限られた範囲にのみ共有**して運用します。
- 回答が積まれる Canonical スプレッドシート自体の共有範囲も最小化してください（アプリ用に「閲覧」を開ける場合も、編集権は付けない）。

## 設問を増減したいとき

1. `scripts/forms/publication_form.gs` の `FIELD_MAP` を編集（種別・設問・必須・選択肢）。
   - フィールドを増減する場合は、`schema.py`／`ingest_to_canonical.py`／Canonical のヘッダも合わせて更新します。
   - `tests/test_form_fields.py`（`tests/verify.py` からも実行）で一致を検証できます。
2. `buildForm` を実行して**新しいフォームを作り直し**、新しい公開 URL を配布。
3. 旧フォームは受付終了（設定 →「回答」→「**回答を受付中**」を OFF）にして凍結。

## 補足：このスクリプトがしていること

- `buildForm()`：1 つのフォームを生成し、先頭に「報告者氏名」「業績種別（ラジオ）」を置き、種別の選択でその種別のセクションへ分岐させる。各設問とフィールドの対応（item ID → 種別・論理フィールド）を保存し、`onFormSubmit` トリガを設置する。
- `route(e)`：送信時に起動。選ばれた種別の値だけを集め、Canonical の該当タブのヘッダ順に 1 行追記する（`record_id` 採番・`status=未確認`・`submitter`＝報告者氏名・`source=form`・`created_at`）。タブ名・ヘッダ・`status` 文言は `loader.py`／`schema.py` と一致。
